# 平台规范模板

每个平台对应一套规则，在组装 Compiler Prompt 时将对应规则填入 `{platform_rules}` 占位符。

## Twitter / X 长文

```
【平台规范：Twitter / X】
- 首条推文：必须是金句或强观点，前140字决定展开率
- 段落长度：每段≤3行，行间留白，拒绝密集大段
- 总篇幅：建议 800-2000 字（Twitter 长文模式）
- 数字力：适度使用数字增强说服力（"3个理由"、"97%的人"）
- 互动结尾：最后1-2句引导转发/评论（"你认为呢？"/"转给需要的人"）
- 不用表情符号，风格偏向知识型输出者
- 段落之间用空行分隔
```

## 小红书

```
【平台规范：小红书】
- 标题：18字以内，含数字/疑问/对比，例：「这3个方法让我效率翻倍」
- 开头：1-2句话直接点明读者利益点或痛点，制造共鸣
- 正文：300-800字，每段≤4行，清单体/并列体优先
- 表情符号：每2-3段配1-2个相关 emoji，不要泛滥
- 小标题：用 ▶ 或 📌 引导各部分
- 结尾：引导收藏（"建议先收藏再看！"）+ 轻量互动（"你有没有类似经历？"）
- 话题标签：结尾追加3-5个相关话题（#学习方法 #效率提升）
```

## 公众号

```
【平台规范：公众号】
- 标题：能引发好奇或共鸣，20字以内，避免标题党
- 摘要/导语：前2-3行即点明文章价值，决定打开率
- 结构：用二级小标题（## 或加粗）分节，每节500-800字
- 段落：每段3-5行，有节奏感，避免连续密集长段
- 篇幅：800-3000字，视深度而定
- 金句：每节至少1句可截图分享的核心观点
- 结尾：引发深层思考或行动（"你会如何选择？"）+ 可选的征集互动
- 不强制使用 emoji，视品牌调性决定
```

## 通用长文

```
【平台规范：通用长文】
- 无平台限制，优先服从 Style Profile 的结构习惯
- 建议有清晰的开篇、展开、收束三段结构
- 篇幅自由，但每个论点需有充分展开
- 适合作为完整文章草稿或深度分析报告
```

## LinkedIn / 职场

```
【平台规范：LinkedIn】
- 开头：提出反直觉观点或职场洞察，前2行必须抓眼球（因为需点"更多"）
- 段落：每段1-2行，大量空白，手机端阅读友好
- 语气：专业但不刻板，第一人称分享经历/洞察
- 数据：适当引用具体数据增强可信度
- 结尾：提出开放性问题引导评论
- 总篇幅：600-1200字
```

---

## 如何在 Compiler Prompt 中使用

1. 根据用户指定的平台，从上方选取对应规范文本
2. 将规范文本填入 Compiler Prompt 的 `{platform_rules}` 位置
3. 将平台名称填入 `{platform}` 位置

示例：

```python
platform = "小红书"
platform_rules = """
【平台规范：小红书】
- 标题：18字以内，含数字/疑问/对比
...
"""

compiler_prompt = COMPILER_TEMPLATE.replace("{platform}", platform)
compiler_prompt = compiler_prompt.replace("{platform_rules}", platform_rules)
# ... 继续替换其他占位符
```
